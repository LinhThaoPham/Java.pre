

# Giới thiệu về JAVA

### Ngôn ngữ Java là gì
- Là ngôn ngữ lập trình bậc cao, hướng đối tượng.
- **1991**: Ra đời với tên gọi Oak bởi Sun Microsystem.
- **1995**: Đổi tên thành Java (theo tên hòn đảo trồng cafe ở Indonesia) => logo của Java.
- **2010**: Oracle mua lại.

### Lí do ra đời của Java
- Nhu cầu về tính di động: Java được phát triển với mục tiêu tạo ra một ngôn ngữ lập trình có thể chạy trên nhiều nền tảng khác nhau mà không cần thay đổi mã nguồn. Khẩu hiệu "Write Once, Run Anywhere" (WORA) thể hiện điều này.
- Tính bảo mật: Với sự gia tăng sử dụng Internet, cần có các ngôn ngữ lập trình có tính bảo mật cao hơn. Java cung cấp các tính năng bảo mật mạnh mẽ, bảo vệ ứng dụng khỏi các lỗ hổng.
- Môi trường lập trình hướng đối tượng: Java được thiết kế dựa trên các nguyên tắc lập trình hướng đối tượng, giúp tăng cường khả năng tái sử dụng mã và giúp việc phát triển phần mềm dễ quản lý hơn.
- Đơn giản và dễ học: Java được phát triển để đơn giản hơn so với các ngôn ngữ như C++. Cú pháp của Java dễ hiểu, giúp người mới bắt đầu dễ dàng tiếp cận.
Hỗ trợ đa dạng các ứng dụng: Java có khả năng phát triển nhiều loại ứng dụng, từ ứng dụng di động (Android) đến ứng dụng web và hệ thống lớn.

### Cách Java hoạt động, điều gì xảy ra khi chạy code Java (.java)
Khi bạn viết và chạy mã Java (.java), quy trình hoạt động của nó diễn ra qua nhiều bước. Dưới đây là các bước chính:

1. **Viết mã nguồn**:
   - viết mã Java trong một tệp có đuôi `.java`. Ví dụ: `HelloWorld.java`.

2. **Biên dịch mã nguồn**:
   - Sử dụng trình biên dịch Java (Java Compiler, `javac`), mã nguồn `.java` được biên dịch thành bytecode, lưu trong tệp có đuôi `.class`. 
   - Lệnh: 
     ```bash
     javac HelloWorld.java
     ```

3. **Thực thi bytecode**:
   - Bytecode là một định dạng trung gian độc lập với nền tảng. Để chạy bytecode, bạn sử dụng Java Virtual Machine (JVM).
   - Lệnh:
     ```bash
     java HelloWorld
     ```

4. **JVM và hoạt động**:
   - JVM sẽ thực hiện bytecode bằng cách chuyển đổi nó thành mã máy (machine code) tương ứng với nền tảng cụ thể mà bạn đang sử dụng.
   - JVM có thể sử dụng Just-In-Time (JIT) Compiler để biên dịch bytecode thành mã máy trong thời gian thực, giúp tăng tốc độ thực thi.

5. **Chạy chương trình**:
   - Sau khi bytecode được biên dịch thành mã máy, chương trình sẽ được thực thi. Bạn có thể tương tác với chương trình (nếu nó có giao diện người dùng hoặc nhận đầu vào từ bàn phím).

- Tóm tắt
Quá trình từ mã nguồn đến thực thi của Java bao gồm biên dịch mã nguồn thành bytecode, sau đó thực thi bytecode trên JVM, cho phép Java chạy trên nhiều nền tảng khác nhau mà không cần thay đổi mã nguồn.

### Cấu trúc 1 chương trình Java

- Tên file Java phải trùng với tên của class. Khi lưu file, lưu nó với tên của class và thêm ".java" vào sau.

### Package là gì
Trong Java, **package** là một cách tổ chức và quản lý các lớp (class) và các thành phần khác trong dự án. Dưới đây là một số điểm quan trọng về package:

1. **Định nghĩa**
- Package là một tập hợp các lớp, giao diện và các thành phần khác có liên quan, giúp nhóm chúng lại với nhau theo một cách có cấu trúc.

 2. **Lợi ích của Package**
- **Tổ chức mã nguồn**: Giúp dễ dàng quản lý và tìm kiếm mã nguồn trong dự án lớn.
- **Tránh xung đột tên**: Khi có nhiều lớp có cùng tên từ các package khác nhau, bạn có thể sử dụng tên đầy đủ (bao gồm tên package) để phân biệt.
- **Kiểm soát quyền truy cập**: Có thể xác định quyền truy cập cho các lớp, giúp bảo vệ dữ liệu và chức năng bên trong các package.

 3. **Cách tạo package**
- Để tạo một package, bạn chỉ cần thêm dòng sau ở đầu tệp Java:
  ```java
  package com.example.myapp;
  ```
- Tên package thường được viết dưới dạng chữ thường và được phân tách bằng dấu chấm.

 4. **Sử dụng package**
- Để sử dụng các lớp trong một package khác, bạn có thể sử dụng từ khóa `import`. Ví dụ:
  ```java
  import com.example.myapp.MyClass;
  ```

5. **Các loại package**
- **Package chuẩn**: Bao gồm các package đã được định nghĩa sẵn trong Java như `java.util`, `java.lang`, `java.io`, v.v.
- **Package tùy chỉnh**: Là các package do lập trình viên tự định nghĩa cho dự án của mình.

 Tóm tắt
Package là một công cụ hữu ích trong Java để tổ chức và quản lý mã nguồn, giúp giảm xung đột tên và tăng cường bảo mật cho các lớp và giao diện.
# Syntax cơ bản

### 1. **Khai báo biến nguyên thủy**
- Các kiểu dữ liệu cơ bản (type): `String`, `int`, `float`, `char`, `boolean`.
- Cách khai báo biến: `Kiểu dữ liệu + tên biến;`
- Final Variables = constant Variables (`final + type + name = ...`)

- Nhập biến:
    - `import java.util.Scanner; // Nhập thư viện Scanner`
    - `Scanner scanner = new Scanner(System.in); // Tạo đối tượng Scanner`
    - `int number = scanner.nextInt(); // Nhập số nguyên`
    - `nextInt()` để nhập số nguyên.
    - `nextDouble()` để nhập số thực.
    - `nextLine()` để nhập chuỗi.
    - `next()` để nhập một từ.

### 2. **Làm quen với vòng lặp**
- **Switch**
  ```java
  switch(expression) {
    case x:
      // code block
      break;
    case y:
      // code block
      break;
    default:
      // code block
  }
  ```

- **While**
  ```java
  while (condition) {
    // code
  }
  ```

- **Do/While**
  ```java
  do {
    // code 
  } while (condition);
  ```

- **For**
  ```java
  for (statement1; statement2; statement3) {
    // code block to be executed
  }
  ```
  - Câu lệnh 1 được thực thi (một lần) trước khi thực thi khối mã.
  - Câu lệnh 2 xác định điều kiện để thực thi khối mã.
  - Câu lệnh 3 được thực thi (mỗi lần) sau khi khối mã đã được thực thi.

### 3. **Câu lệnh rẽ nhánh**
```java
if (condition1) {
  // block of code to be executed if condition1 is true
} else if (condition2) {
  // block of code to be executed if condition1 is false and condition2 is true
} else {
  // block of code to be executed if condition1 is false and condition2 is false
}
```

### 4. **Mảng trong Java**
- Mảng có sẵn giá trị: `type[] name = {value1, value2, value3, ...};`
- Mảng nhập giá trị: `type[] name = new type[size];`
- Mảng 2 chiều:
  ```java
  type[][] name = { {1, 2, 3, 4}, {5, 6, 7} };
  type[][] arrayName = new type[rows][columns];
  ```

# Tổng quan về Class và Object
### **this**
- Từ khóa `this` là một tham chiếu (reference) đặc biệt đến đối tượng hiện tại của lớp mà nó được gọi. Nó được sử dụng trong các phương thức hoặc constructor của lớp để phân biệt giữa các biến instance của đối tượng và các tham số của phương thức hoặc constructor khi chúng có cùng tên.

```java
public class Main {
  int x;

  // Constructor with a parameter
  public Main(int x) {
    this.x = x;
  }

  // Call the constructor
  public static void main(String[] args) {
    Main myObj = new Main(5);
    System.out.println("Value of x = " + myObj.x);
  }
}
```
### **Constructor**
- Hàm khởi tạo.
- Dữ liệu nên được khởi tạo trước khi sử dụng.
- Phương thức khởi tạo là phương thức đặc biệt được gọi tự động ngay sau khi đối tượng được tạo ra nhằm mục đích chính là khởi tạo cho các thuộc tính của đối tượng.
  - Tên trùng với tên lớp.
  - Không nhận giá trị trả lại.
  - Được gọi tự động mỗi khi đối tượng được tạo ra bởi toán tử `new`.
  - Nếu không khai báo, hệ thống sẽ gọi phương thức khởi tạo mặc định - phương thức rỗng.
### **Access Modifier**

1. `public`:
   - Bất kỳ lớp nào cũng có thể truy cập được từ bất cứ đâu.
   - Có thể áp dụng cho lớp, phương thức và biến.
   ```java
   public class Person {
     public String name;
     public void displayName() {
       System.out.println(name);  // Phương thức công khai
     }
   }
   ```

2. `private`:
   - Chỉ có thể truy cập từ bên trong lớp chứa nó.
   - Thường được sử dụng để bảo vệ dữ liệu và ngăn không cho các lớp khác can thiệp trực tiếp.
   ```java
   public class Person {
     private String name;  // Chỉ có thể truy cập trong lớp Person
     public void setName(String name) {
       this.name = name;  // Phương thức công khai để thay đổi giá trị name
     }
     public String getName() {
       return name;  // Trả về giá trị của name
     }
   }
   ```

3. `protected`:
   - Cho phép truy cập trong cùng một gói và các lớp con (ngay cả khi lớp con nằm ở gói khác).
   - VD:
   ```java
   public class Person {
     protected String name;  // Có thể truy cập trong gói và lớp con
     protected void displayName() {
       System.out.println(name);  // Phương thức có thể truy cập bởi lớp con
     }
   }
   ```

4. Mặc định (default): Nếu không chỉ định một Access Modifier, Java sử dụng mặc định.
### **getter**
**Getter** là một phương thức trong lập trình hướng đối tượng, thường được sử dụng để truy cập giá trị của các thuộc tính (hoặc biến) riêng tư trong một lớp (class). Getter giúp kiểm soát việc truy cập và cung cấp một cách an toàn để lấy giá trị của thuộc tính mà không cho phép thay đổi giá trị của nó từ bên ngoài.

 1. **Cách hoạt động**
- Getter thường có tên bắt đầu bằng từ "get" theo sau là tên của thuộc tính, với chữ cái đầu tiên của tên thuộc tính được viết hoa.
- Phương thức này trả về giá trị của thuộc tính mà nó đại diện.

 2. **Ví dụ trong Java**
Dưới đây là ví dụ về cách sử dụng getter trong một lớp Java:

```java
public class Person {
    // Thuộc tính riêng tư
    private String name;

    // Constructor
    public Person(String name) {
        this.name = name;
    }

    // Getter cho thuộc tính name
    public String getName() {
        return name;
    }
}

// Sử dụng lớp Person
public class Main {
    public static void main(String[] args) {
        Person person = new Person("Alice");
        // Gọi getter để lấy tên
        System.out.println(person.getName()); // In ra: Alice
    }
}
```

 3. **Lợi ích của Getter**
- **Bảo mật**: Giữ cho thuộc tính riêng tư, chỉ cho phép truy cập thông qua getter.
- **Kiểm soát**: Có thể thêm logic vào getter (ví dụ: kiểm tra điều kiện) trước khi trả về giá trị.
- **Tính nhất quán**: Giúp đảm bảo rằng giá trị được truy cập một cách nhất quán và an toàn.

Tóm tắt
Getter là một phương thức quan trọng trong lập trình hướng đối tượng, giúp truy cập các thuộc tính riêng tư một cách an toàn và kiểm soát.
### **setter**

**Setter** là một phương thức trong lập trình hướng đối tượng, thường được sử dụng để thiết lập giá trị cho các thuộc tính (hoặc biến) riêng tư trong một lớp (class). Setter giúp kiểm soát việc thay đổi giá trị của thuộc tính và có thể bao gồm logic bổ sung để đảm bảo rằng giá trị được thiết lập là hợp lệ.

 1. **Cách hoạt động**
- Setter thường có tên bắt đầu bằng từ "set" theo sau là tên của thuộc tính, với chữ cái đầu tiên của tên thuộc tính được viết hoa.
- Phương thức này nhận một tham số, thường là giá trị mới mà bạn muốn gán cho thuộc tính, và không trả về giá trị nào.

 2. **Ví dụ trong Java**
Dưới đây là ví dụ về cách sử dụng setter trong một lớp Java:

```java
public class Person {
    // Thuộc tính riêng tư
    private String name;

    // Constructor
    public Person(String name) {
        this.name = name;
    }

    // Setter cho thuộc tính name
    public void setName(String name) {
        this.name = name;
    }

    // Getter cho thuộc tính name
    public String getName() {
        return name;
    }
}

// Sử dụng lớp Person
public class Main {
    public static void main(String[] args) {
        Person person = new Person("Alice");
        
        // Gọi setter để thiết lập tên mới
        person.setName("Bob");
        
        // Gọi getter để lấy tên
        System.out.println(person.getName()); // In ra: Bob
    }
}
```

 3. **Lợi ích của Setter**
- **Bảo mật**: Giữ các thuộc tính riêng tư, chỉ cho phép thay đổi thông qua setter.
- **Kiểm soát**: Có thể thêm logic vào setter để kiểm tra giá trị trước khi gán (ví dụ: kiểm tra xem giá trị không rỗng).
- **Tính nhất quán**: Giúp đảm bảo rằng giá trị được thiết lập một cách hợp lệ và an toàn.

Tóm tắt
Setter là một phương thức quan trọng trong lập trình hướng đối tượng, giúp thiết lập giá trị cho các thuộc tính riêng tư một cách an toàn và kiểm soát.
